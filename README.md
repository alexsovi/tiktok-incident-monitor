# Детектор Авариек (TikTok Mods Incident Monitor Bot)

Асинхронный Telegram-бот для автоматического мониторинга конфигурационных файлов на "аварийки" сторонних модификаций TikTok. 

Бот отслеживает активацию принудительных блокировок приложения (так называемых "аварийных обновлений" и "серых баннеров") и автоматически рассылает уведомления подписчикам.

**Примечание:** Код данного проекта почти полностью является вайбкодом.

## Основные возможности

- **Асинхронный парсинг:** Проверка `.conf` файлов и `.txt` файлов с использованием `httpx` и `aiogram`.
- **Автоматическое сканирование (Auto-discovery):** Бот самостоятельно находит новые конфиги на аварийки (например, `banner3.conf`), уведомляет об этом и обновляет счетчики в базе данных.
- **Мониторинг 4 категорий:**
  - Обычные аварийки
  - "Анти"-аварийки
  - Серые баннеры
  - "Анти"-серые баннеры
- **Защита от ложных срабатываний:** Порог подтверждения (`CONFIRM_THRESHOLD = 3`) для фиксации окончания инцидента.
- **Поддержка Proxy:** Маршрутизация трафика Telegram API через прокси.
- **Готовая история инцидентов:** Бот использует существующую (неполную) базу `incidents.db` для сохранения хронологии блокировок.

## Технологический стек

- Python 3.8+
- aiogram >= 3.0.0
- httpx >= 0.25.0
- python-dotenv >= 1.0.0
- SQLite3

## Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/alexsovi/tiktok-incident-monitor.git
   cd tiktok-incident-monitor
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения:**
   Создайте файл `.env` в корневой директории и добавьте:
   ```env
   TELEGRAM_BOT_TOKEN=ваш_токен_бота
   PROXY_URL=http://ваш_прокси (опционально)
   ```

4. **Базы данных:**
   В репозитории уже присутствует файл `incidents.db` с собранной историей. Бот продолжит запись в него. Файл `subscribers.db` создастся автоматически.

5. **Запуск:**
   ```bash
   python main.py
   ```

## Команды бота

* `/start` — Информация о боте и команды.
* `/status` — Текущее состояние мониторинга всех конфигов.
* `/history` — История последних 50 инцидентов (опирается на `incidents.db`).
* `/yes` — Подписаться на рассылку уведомлений об аварийках.
* `/no` — Отписаться от рассылки.

## Структура баз данных

1. **`incidents.db`**
   - Таблицы: `incidents`, `incidents_anti`, `incidents_grey`, `incidents_anti_grey` (`id`, `start_time`, `end_time`, `urls`).
   - Таблица: `settings` (счетчики файлов: `max_banner_index`, `max_anti_index`).
2. **`subscribers.db`**
   - Таблица: `subscribers` (`user_id`, `subscribed`).
